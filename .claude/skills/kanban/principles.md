# 대원칙: 안전 우선 실행 원칙

**속도는 목표가 아니다. 제대로 하는 것이 목표다.**

---

## 코드베이스 우선 탐색 (계획 전 필수)

- 추정하지 말 것 — 계획하기 전에 반드시 관련 파일과 기존 패턴을 직접 확인
- 기존 인터페이스, 파일 구조, 의존성을 파악한 뒤 범위 확정
- "아마 이렇게 되어있겠지"는 금지 — 코드에서 확인한 사실만 계획에 반영

## 카드 분리 기준

아래 중 하나라도 해당하면 카드를 분리한다:

- 구현 예상 시간이 1시간 초과
- 두 개 이상의 레이어(DB / 서비스 / UI 등)에 걸친 변경
- 실패 시 롤백이 복잡한 경우
- 요구사항 중 불확실한 항목이 1개 이상

## 진행 전 체크

- 이 카드의 완료 조건이 명확하고 검증 가능한가?
- 이 카드가 실패해도 다른 카드에 영향을 주지 않는가?

## 금지 행동

- 리파인 없이 바로 구현 시작
- "일단 해보고 나중에 정리" 식의 진행
- 카드 범위를 확장해가며 구현
- 코드베이스 확인 없이 추정으로 계획 수립
